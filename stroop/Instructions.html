In this task, you will see color words (like Red, Blue, Green, Yellow) and filled circles (•) displayed in different font colors.

Your task: Identify the color of the circle or the font color of the word, respectively—not the word itself.

Examples

• → Correct answer: Green (color of the circle)

Red → Correct answer: Red (word and color match)

Red → Correct answer: Blue (ignore the word, focus on the font color)

Located around the word, you will see several buttons with color names.

A correct response is made by pressing the button that names the color of the circle or font color of the word, respectively.

Examples

Yellow Red Blue Green

• → The correct response is pressing the Green button, since the circle is displayed in green.

Red → The correct response is pressing the Red button, since the word is displayed in red.

Red → The correct response is pressing the Blue button, since the word is displayed in blue.

Respond as quickly and accurately as possible.

Remarks for experimenters regarding the implementation

  • The test includes a neutral (non-word) condition (filled circles: •) as a baseline and to screen for color blindness.
  • Each trial starts with pressing a “Next” button. The response buttons are placed in approximately equal distance around the position of this “Next” button so that each response button is approximately equally easy to reach.
  • With each trial, the meaning of the response buttons changes randomly. This is to prevent learning of the buttons’ locations. Such learning might speed up responses over time, and a time trend may pose problems for statistical analysis, in particular, with relatively few iterations. (For instance, when there are more congruent trials towards the end, a Stroop effect in the response times would be overestimated if responses become faster towards the end as a result of learning.)
  • The response buttons must be sufficiently easy to hit. That is, they must not be too far from the center (the “Next” button), and they must be sufficiently large. This is because otherwise, participants will need so much time to respond that they start deliberating while responding, thereby potentially wiping out any Stroop (word interference) effect.
  • The font sizes and the size of the response buttons adapt to the viewport height and width to make the test universally usable across a wide range of screen sizes and aspect ratios.
  • The share of congruent trials among all trials is drawn randomly (by default, between 40% and 60%). This is because a fixed share may enable participants to anticipate the type of the upcoming iteration(s). (Say, if I knew that there were 10 trials of which exactly 50% are congruent, then I could anticipate all remaining trials to be incongruent, once I have completed 5 congruent trials.)
